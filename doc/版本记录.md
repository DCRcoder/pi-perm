# 版本记录

## 2026-07-04

- SRT settings 默认改为写入 pi-perm extension 数据目录 `~/.pi/agent/extensions/pi-perm/runtime/`，不再在当前项目目录创建或使用 `runtime/`。
- `runtime.settingsDir` 只允许相对路径，并解析到 `runtime.baseDir` 下；绝对 `settingsDir` 会被配置校验拒绝。
- 新增 `runtime.sessionAllowTtlMs`，session 级授权会按空闲 TTL 自动清理，过期后重新请求确认。
- 新增确认选项：命中 `confirm` 规则时支持“拒绝 / 允许一次 / 本 session 始终允许”。
- `本 session 始终允许` 只在当前 Pi session 内通过内存缓存生效，不写入配置；授权范围按 profile、工具、规则或操作 ID、目标摘要限定。
- 切换 profile 时清空 session 授权缓存，避免旧 profile 授权影响新 profile。
- 补充 session 授权测试，覆盖重复命中跳过确认、不同目标仍需确认、profile 切换缓存清理和旧 `ctx.ui.confirm` 兼容行为。
- 等待用户处理权限确认期间，通过 Pi UI 状态 API 显示 blocked 风格状态，并在确认完成或异常后恢复默认 working 状态。
- 已安装 Herdr Pi integration 时，等待权限确认期间通过 Pi event bus 发送 `herdr:blocked` active true/false，让 Herdr 可显示 blocked 状态。
- 明确 Herdr `done` 由 idle/unseen 派生，pi-perm 释放 blocked 状态后交由 Herdr 自行显示 done 或 idle。

## 2026-07-02

- 新增 PiPerm 权限控制实现文档，说明 Pi extension 工作原理、当前项目生效流程、配置链路、工具拦截流程和 Mermaid 流程图。
- 将 `core/` 与 `test/` 从 JavaScript 迁移为 TypeScript，测试脚本改为通过 `tsx` 运行。
- 将操作权限配置调整为原命令模式优先，例如 `git push`、`rm -r`、`curl | sh`。

## 背景

pi-perm 当前默认把 SRT settings 写入项目当前目录的 `runtime/`。该目录属于扩展运行时状态，不属于被保护项目的源码或业务产物；在项目根目录生成它会污染工作区、干扰开发和版本控制。

session 级授权目前只存在于内存集合中，虽然不会写入配置，但没有空闲过期机制。长时间不再使用同一 session 后，授权仍会在 extension 实例存活期间继续命中。

## 目标

- 所有 pi-perm 运行时状态默认写入 pi-perm 自己的 extension 数据目录。
- SRT settings 不再写入当前项目目录的 `runtime/`。
- 不再保留“相对 runtime 目录解析到当前项目目录”的机制。
- session 级授权支持空闲超时，过期后自动清理并重新请求确认。
- 保留用户配置能力，但配置路径必须解析到 extension 数据目录或用户显式指定的绝对路径。

## 非目标

- 不改变权限规则匹配、确认弹窗选项或 SRT settings 内容结构。
- 不持久化 session 授权。
- 不迁移或删除历史已经生成的项目根 `runtime/` 目录。
- 不改变审计日志默认文件位置。
